## 2. Vercel 前端部署（已完成）

### Dashboard 配置（必须保持）

| 设置 | 值 |
|------|-----|
| Root Directory | **`frontend`** |
| Framework Preset | Vite |
| Build Command | `npm run build` |
| Output Directory | `dist` |

### 关键文件：`frontend/vercel.json`（Root Directory=frontend 时生效）

```json
{
  "rewrites": [
    {
      "source": "/api/:path*",
      "destination": "https://YOUR_BACKEND_URL/api/:path*"
    },
    {
      "source": "/uploads/:path*",
      "destination": "https://YOUR_BACKEND_URL/uploads/:path*"
    },
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

**规则顺序很重要：**

1. 先 `/api`、`/uploads` 代理到后端  
2. 最后 SPA 回退到 `index.html`（解决 `/admin` 等 history 路由 404）

**静态资源安全：** Vercel 先匹配 `dist` 真实文件，再 rewrite，因此 `/assets/*` 不会被当成 SPA。

### 仓库根目录 `vercel.json`

- 内容与 `frontend/vercel.json` 对齐（占位 `YOUR_BACKEND_URL`）
- **当前 Root Directory = frontend 时根目录配置不生效**
- 若以后把 Root Directory 改成仓库根，必须以根目录这份为准，且同样保证 API rewrite 在 SPA 之前

## 4. 登录链路（排查用）

```
浏览器 → POST https://hanx33.vercel.app/api/admin/login
       → Vercel rewrite → https://YOUR_BACKEND_URL/api/admin/login
       → Spring Boot AdminAuthController
       → 校验 H2 中 BCrypt 密码 → 返回 JWT
```

### 前端关键代码

| 文件 | 作用 |
|------|------|
| `frontend/src/api/index.js` | `baseURL: '/api'`；`adminApi.login` → `POST /admin/login`；Bearer token |
| `frontend/src/views/admin/Login.vue` | 成功写 `localStorage.admin_token`；网络失败单独提示 |
| `frontend/src/router/index.js` | `createWebHistory()`；`/admin` 需 token，否则去登录页 |
| `frontend/vite.config.js` | **仅本地 dev**：`/api`、`/uploads` 代理到 `localhost:8080` |

### 后端关键代码

| 文件 | 作用 |
|------|------|
| `AdminAuthController` | `POST /api/admin/login`；错密返回 body `code:401`（HTTP 仍可能是 200） |
| `DataInitializer` | 仅当 admin 表为空时创建 `admin`/`admin123` |
| `SecurityConfig` | `/api/admin/login` permitAll；其余 `/api/admin/**` 需 JWT |
| `CorsConfig` | 允许 `https://hanx33.vercel.app`、`http://localhost:5173`、`127.0.0.1:5173` |
| `application.yml` | 端口、`jwt.*`、H2、`file.upload-dir` |
| `JwtUtil` + `JwtAuthenticationFilter` | 签发/校验 Bearer token |

### 常见失败表现

| 现象 | 原因 |
|------|------|
| 提示「无法连接服务器…」 | 后端未部署，或 vercel 代理仍是 `YOUR_BACKEND_URL` |
| 提示「用户名或密码错误」 | 后端可达，但账号错或库里没有该 admin |
| `/admin` 404（HTML） | SPA rewrite 未生效 / Root Directory 不对 |
| 登录 200 但进不了后台 | token 未写入或路由守卫异常 |

**注意：** 后端业务失败常用 **HTTP 200 + body.code**；axios 拦截器只按 **HTTP 401** 清 token 跳转。登录错密不会误跳登录页。

## 5. 后端如何部署（不要用 Vercel）

需要 **Java 17 + 可执行 JAR** 的平台。

### 推荐：Render（免费档可演示）

1. https://render.com → 用 GitHub 登录  
2. New → Web Service → 选 `hancc33/hanc666`  
3. 配置：

| 项 | 值 |
|----|-----|
| Root Directory | `backend` |
| Runtime | Java |
| Build | `mvn clean package -DskipTests`（或 `./mvnw ...` 若有 wrapper） |
| Start | `java -jar target/zhumengmiao-backend-1.0.0.jar` |
| Plan | Free |

4. 部署后记下 URL，例如 `https://xxxx.onrender.com`  
5. 自检：`GET https://xxxx.onrender.com/api/public/videos` 应返回 JSON

### 备选

- **Railway**：Root=`backend`，Maven 构建  
- **Fly.io**：需 Dockerfile  
- **云主机 / 学校服务器**：`java -jar` + Nginx HTTPS  
- **临时演示**：本机 `mvn spring-boot:run` + ngrok/cpolar，公网 HTTPS 填进 vercel.json  

### 部署注意

1. **H2 与 uploads 在本地磁盘**：PaaS 重启/重部署可能丢数据；演示够用，正式建议 MySQL + 对象存储  
2. **首次空库** 会创建 admin/admin123；已有库则不会重复初始化  
3. Vercel rewrite 目标建议用 **HTTPS 公网域名**  
4. 云平台会注入 `PORT`，`application.yml` 已写 `port: ${PORT:8080}`  
5. 大文件上传 multipart 上限 yml 已设 500MB，平台可能另有限制  

### 本地打包运行

```bash
cd backend
mvn clean package -DskipTests
java -jar target/zhumengmiao-backend-1.0.0.jar
```

登录接口自测：

```http
POST http://localhost:8080/api/admin/login
Content-Type: application/json

{"username":"admin","password":"admin123"}
```

## 6. 后端上线后：接到 Vercel 的步骤

1. 确认后端公网 HTTPS 可访问（至少 `/api/public/videos`）  
2. 编辑 **`frontend/vercel.json`**（Root Directory=frontend 时只改这份即可）：  
   把 `YOUR_BACKEND_URL` 换成真实主机（两处：`/api` 与 `/uploads`）  
3. 可选：同步改仓库根 `vercel.json`，避免以后改 Root Directory 时踩坑  
4. `git commit` + `git push` → 等 Vercel 自动部署  
5. 打开 https://hanx33.vercel.app/admin/login ，用 admin / admin123 登录  

走 Vercel rewrite 时浏览器请求是同源 `/api/...`，通常**不触发跨域**；CORS 用于直连后端或预检兜底。

## 8. 约束与约定（给后续 AI / 开发者）

1. **本次/联调阶段**：优先最小改动；不要大改业务代码  
2. **不要改默认管理员密码**，除非用户明确要求
